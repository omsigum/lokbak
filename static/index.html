A quick introduction

This API is for note keeping, Add a note and remember all the stuff you have to get done.

Adding a user

Getting started, to add a user it really is simple, you are going to manipulate the /user/ for user a user action. (note: all user actions are behind the /user/) the full path is then /user/add/username/fullname/securepassword, an example of this request is /user/add/jhon/Jhon Doe/JhonDoessecurepass.

Getting a live api key

The password is only a mechanism for a user to authenticate. All the real interaction goes through the apiKey. There are a couple of reasons for this, The api key only lasts 3 hours from last use. so the user after a 3 hour period of inactivety will have regenerate a valid api key. second, this raises the complexity of the authntication part which i found amusing. To get a valid api key you use the /user prefix with the addon of /getkey/username/password, now you can run this as many times as you want. The system only genereates a new key after the old one is invalid.

Adding a note

Adding a note can be done with the key, simply call /note (note: for note action) with the addon of /add so the entire request will look like /notes/add/content/apikey

List all the users notes and content

To view all your notes run /notes/ls/apikey. simple as that

Delete a note

WARNING this action is permanent. To delete a note we must refernce the call above. from that we get amongst other things the noteid. That is used for refernce when a note is deleted. to delete a note. run /note/del/noteid/apik.

Archive a note

Archiving a note is not deleting it. you can still list and even "bring back to life" your archived notes with a simple API call, (demonstrate below) call /notes/arc/noteid/apik

List archived notes

To view archived notes run /notes/lsarc/apik

Unarchive a note

As we can archive we can unarchive. To unarchive you must recive the id of the note you want to unarchive from the list archived query above. Then run /notes/unarc/noteid/apikey.
